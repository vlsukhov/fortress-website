We Make Decentralized Finance Secure

Fortress is a full-stack security solution specifically developed for decentralized finance (DeFi) apps. It’s real-time, decentralized and permissionless.

What is Fortress?

The current security solutions in DeFi are static smart contract audits, which are effective in finding vulnerabilities before the DeFi app goes live. However, once the app goes live and interacts with unaudited contracts, the whole static audit becomes almost obsolete. Add that "trivial" code updates that usually get deployed unaudited, any vulnerability becomes open to manipulations by malicious actors. As Fortress Foundation we examined every security incident related to DeFi in 2019 and 2020. As a result we developed and open sourced 3 different solutions that work together to allow DeFi developers to sustainably build applications which secure high value assets. Fortress Live Security Audit Engine which does real-time security audits before smart contract transactions. Fortress Threat Detection and Response Engine which monitors DeFi app protocols to detect malicious actors and give appropriate responses. Fortress Time-wrapped Price Feed Oracle that provides prices in a decentralized, permissionless and manipulation-resistant way.

Our Features

2 engines. 1 oracle. Working in conjunction with each other. Third-party-free, open-sourced and community-driven.

Real-time Security Audit Engine

Fortress Real-time Security Audit Engine does a live automatic security audit before every transaction that includes write operation to the smart contracts or any interaction with other protocols. It assumes that the code already has a static security audit so it maintains the validity of that audit.

Threat Detection and Response Engine

Fortress Threat Detection and Response Engine uses anomaly detection and machine learning to monitor DeFi app’s streaming metrics in real-time. It collects pair, contract and transaction metrics from on-chain sources and finds threat patterns in data for the response mechanism to act-upon.

Time Wrapped Price Feed Oracle

Fortress Time-Wrapped Price Oracle is an on-chain price feed oracle that interacts with Uniswap v2 in a trustless and decentralized way. Through the use of cumulative price-time values, it is expensive to manipulate over small periods of time, and impossible to manipulate within a single transaction.

Frequently Asked Questions

We're here to help. Contact us if you have any more questions. Read our whitepaper to have a deeper understanding of what we do.

As we’ve seen from the most recent bZx, Harvest, Pickle Finance and Compounder hacks; getting audited from a security audit firm certainly doesn’t guarantee that a DeFi app or protocol that it’s connected to will be completely free from security issues. Think about your house. If you want to protect your house from theft, buying a strong door before you start living in the house is necessary. However, in order to really protect your house from theft; you need an alarm system and a guard on the door. While static security audit is just a “strong door”, Fortress is an “alarm system” and a “guard” for the DeFi app. It does real-time security audits rather than doing static audits, as other solutions. It monitors the DeFi app for threats and gives appropriate responses, rather than doing nothing, as other solutions. It provides an integrated, decentralized, trustless and manipulation-free price oracle while rather than being a trusted third party, as other solutions. All these make Fortress different.

Fortress is specifically for built for DeFi apps. DeFi app developer teams can integrate Fortress to their tech stacks. While other dApps can leverage it as well, we only want focus on decentralized finance which already needs more care than any other area.

Fortress is a better solution to solve hacking incidents of DeFi protocols than any other solution in the market. This is because it gives direct care to the running application which causes the incidents in the first place. In addition to that, experience from past security incidents are builtin to its AI as threat patterns. As the time goes on and new incidents happen, it will get better and better. While it’s a fact that if something can be exploited, it will be exploited; Fortress acts on-time to decrease that risk to minimum.
